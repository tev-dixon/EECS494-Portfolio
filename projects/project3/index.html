Final Fumes

A First Person 3D Horror Driving game; developed for EECS 494 at the University of Michigan.

About the Project

This game was created for the EECS 494 class project at the University of Michigan. This is my third shot at making a Unity game; it was completed with a small team in just a few weeks while juggling four other courses and finals.

My part on the team was...

  • Creating the procedural generation system including both world chunks and inter-chunk clutter.
  • Creating the advanced internal editor tooling for the procedural generation system.
  • Making many of the technical art decisions.
  • Creating some of the art including roads, some of the signs, and the warped monster.
  • Creating the main lobby (sourcing music, setting up the cinemachine camera, making the camera swap angles on music beats, button scripts, scene set up, art, etc.).
  • Creating the monster behavior and spawning systems.
  • Creating the radio audio using ElevenLabs AI.
  • Creating the radio navigation system.
  • Creating the audio manager system.
  • Creating the final iteration of the damage system.
  • Creating the hints/tooltips system.
  • Creating the events and triggers system.
  • Making a persistant scene transition manager with a cache to store data on scene reload.

The game's primary complexities were:

  • Guiding the player to understand the radio mechanic.
  • Making a modular procedural generation system that has robustness, variety, and both local and global constraints.
  • Balancing monsters and difficulty for the wide variety of skill levels.
  • Making the UI integrated seamlessly into the world.
  • Creating a balanced shop system with a currency, repairs, and upgrades.
  • Making a persistant scene transition manager with a cache to store data on scene reload.

Technologies Used:

  • Unity Game Engine
  • C# Programming Language
  • MagicaVoxel Software
  • Python Programming Language (scripts to create in-game assets and audio)
  • Audacity Software
  • Unity Version Control (Formerly Plastic SCM)
  • Jira for Project Management
